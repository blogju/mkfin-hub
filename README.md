# mkfin-hub

## 项目简介

`mkfin-hub` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常包含特定主题的静态内容或工具型应用，不针对任何单个域名或具体网站。本仓库旨在提供一种轻量、可维护的方式，将分散的 HTML 页面集中托管，便于检索与访问。

## 目录结构

```
mkfin-hub/
├── pages/            # 存放所有发布的 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/           # 公共资源文件（CSS、JS、图片等）
│   ├── style.css
│   └── script.js
├── README.md         # 本文件
└── index.html        # 可选：页面索引或跳转页
```

- **pages/**：每个独立 HTML 页面放置在此目录下，文件名应反映页面用途。
- **assets/**：存储页面可能引用的通用样式、脚本或静态资源。
- **index.html**：可作为入口页面，列出所有已归档的页面链接（可选）。

## 页面归档说明

- 所有归档的 HTML 页面均为静态内容，无需后端支持。
- 每个页面独立完整，不依赖外部服务或特定环境。
- 归档页面可能包含工具、信息展示或纯粹的功能性界面，用途各异。
- 页面更新或新增时，仅需添加或修改对应文件，无需调整仓库整体结构。

## 维护说明

- 维护者应确保每个 HTML 页面可正常渲染，无损坏链接或缺失资源。
- 建议在提交前对页面进行基本测试，包括跨浏览器兼容性检查。
- 若页面引用了 `assets/` 下的资源，请使用相对路径（如 `../assets/style.css`）以保证可移植性。
- 欢迎提交 Issue 或 Pull Request 报告问题或贡献新页面，但请确保内容符合通用规范。

## 使用方式

本仓库中的 HTML 页面可通过 GitHub Pages 或其他静态托管服务直接访问。具体部署方式取决于用户需求，仓库本身不限定访问路径。
